Background
What is the DPDP Act 2023?
India's Digital Personal Data Protection Act, 2023 — the country's first comprehensive data privacy law
📋
Purpose Limitation
Collect data only for a specific, clear purpose. Don't use it for anything else.
🤝
Informed Consent
Data principals must give clear, informed consent before their data is processed.
🔒
Security Safeguards
Data fiduciaries must implement reasonable security measures to protect personal data.
📉
Data Minimization
Collect only what is necessary. No excessive data collection.
🗑️
Right to Erasure
Data principals can request deletion of their personal data.
🚫
No Unauthorized Sharing
Personal data cannot be sold, shared, or disclosed without explicit consent.
Our Commitment
HealQR's Three Absolute Rules
Hardcoded into every layer — server, client, AI, and analytics
🚫
ZERO Data Selling
Patient data is never sold, shared, or mined. Not to pharma, not to insurers, not to anyone.
⚕️
ZERO Treatment Interference
HealQR is a digital booking platform only. We never influence, suggest, or modify medical treatment.
🤖
ZERO AI Medical Advice
Our AI chatbot is restricted server-side from giving medical advice or revealing patient data.

Section 8(3) — Data Sharing Restrictions
Bidirectional Data Masking
Doctors and clinics can restrict data visibility from each other
👨‍⚕️ → 🏥
Doctor Restricts Clinic
Doctor can self-restrict their QR booking data from specific clinics.

When restricted: Patient names show "S***", phone numbers show "******7890".

Clinic sees booking count but not patient identity.
🏥 → 👨‍⚕️
Clinic Restricts Doctor
Clinic can toggle per-doctor data restriction for QR bookings.

When restricted: Masked in Today's Schedule, Reports, and Data Export.

Doctor retains access to their own booking data.
Data Export Masking
Excel exports auto-mask restricted patient fields
Report Masking
maskName() + maskPhone() at display level
Schedule Masking
Today's Schedule masks restricted booking names

Section 8(4) — Database-Level Enforcement
Firestore Security Rules
Access control enforced at the database layer — not just the UI
Owner-Only Subcollections
Doctor data (prescriptions, revenue, receipts, inventory, staff, broadcasts) is locked to isOwner(doctorId).

No other doctor, clinic, or admin can read another doctor's private subcollections.
Three-Tier Clinic Access
Clinic subcollections check:
1. Clinic Owner (UID match)
2. Branch Manager (email in locationEmails)
3. Master Access (masterAccessEmail match)

No other user can access clinic data.
Immutable Analytics
rxMoleculeData and rxExtractionLogs are append-only.

allow update, delete: if false — data integrity guaranteed. No tampering possible.
No Core Data Deletion
allow delete: if false on: bookings, doctors, qrPool, appointments, transactions.

Prevents unauthorized data destruction — accidental or malicious.
